Brio

Behaviour that lives in your HTML.

Brio is a lightweight JavaScript framework for multi-page sites. Bind actions to elements with data-attributes, declare conditions directly in markup, and let the framework handle the wiring. No build step. No reactive state. Just HTML that does what it says.

<button data-click="addToCart(sku-912)">
  Add to cart
</button>

<div data-show-when="delivery==express">
  <label>Delivery time</label>
  <input type="text" id="slot">
</div>

function addToCart(el, event, target) {
}

Why Brio

01

Declare conditions.
Don't write handlers.

Show, hide, enable, or disable any element based on what the user selected — declared directly in HTML. data-show-when, data-hide-when, data-enable-when, data-disable-when. Brio evaluates them on every change. No event listener, no function, no wiring.

<select id="account-type">
  <option value="personal">Personal</option>
  <option value="business">Business</option>
</select>

<div data-show-when="account-type==business">
  <label>VAT number</label>
  <input type="text" id="vat">
</div>

<button type="submit"
  data-disable-when="plan==none">Continue</button>
02

Dynamic binding.
Declared, not coded.

Point data-bind at a field with #id, or at any JavaScript object with dot notation. Mix both in a single template. Brio keeps every reference in sync — summaries, prices, translations — without a single event listener.

<button data-click="decrement(nights)">−</button>
<input id="nights" type="number" data-bind="{{#nights}}">
<button data-click="increment(nights)">+</button>
03

HTML declares.
JavaScript defines.

The attribute names the action. Your function in project.js handles it. No logic in attributes, no markup in scripts — both sides stay readable on their own.

<button data-click="saveItem(draft)">Save draft</button>

function saveItem(el, event, target) {
}
04

Accessible by design.
Not by effort.

Brio uses ARIA as its operating language. aria-modal activates the focus trap. aria-expanded declares toggle state. Form errors announce as alerts. Correct semantics and working behaviour are the same thing.

<div id="confirm" aria-modal="true" hidden>
  <h2>Delete this item?</h2>
  <button data-click="closeDialog">Cancel</button>
  <button data-click="confirmDelete">Delete</button>
</div>

<button data-click="toggle" aria-expanded="false">
  Details
</button>
05

No dependencies.
No bundler. No build step.

Brio is plain JavaScript files. Drop them into any project and they work. No toolchain to configure, no packages to maintain, no version conflicts to debug. Ships exactly as written.

npm install webpack.config.js package.json node_modules transpilation 
<link rel="stylesheet" href="brio.css">
<script defer src="brio.js"></script>
<script defer src="project.js"></script>
