Brio

Behaviour that lives in your HTML.

A tiny JavaScript framework for multi-page sites. Declare actions and conditions in markup. No build step, no dependencies.

Examples of Brio in HTML

  • One attribute, endless possibilities.

    Bind any element to a function. Pass an id or slug in parentheses — your .js files receives it as target.

    Docs → 
    <button data-click="addToCart(sku-912)">Add to cart</button>

  • Declare conditions, don't write event listeners.

    Automatically show, hide, enable, or disable based on another field's value.

    Docs → 
    <div data-show-when="account-type==business">
  <label>VAT number</label>
  <input type="text" id="vat">
</div>

  • Accessible dialogs, out of the box.

    Open a modal; run multiple actions in order with |. Focus trap and Escape are built in.

    Docs → 
    <button data-click="openDialog(confirm)">Delete</button>
<button data-click="deleteItem|closeDialog(confirm)">Confirm</button>

  • Dynamic binding. Declared, not coded.

    Use data-bind to mirror a field (#id) or state/object value. Mix both in {{placeholders}} for live summaries and translations.

    Docs → 
    <span data-bind="{{#nights}} nights · {{pricing.currency}}{{#total}}"></span>

  • Inline feedback. For actions, fields and forms.

    Field-level and form-level messages. showMessage('email', 'Invalid.') or readServerMessages(el, data) — ARIA wired automatically.

    Docs → 
    <input id="email" type="email">
<span data-message data-for="email" hidden></span>
